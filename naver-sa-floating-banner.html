Design Approach
2개 카테고리 구성: "무엇이 달라지는가"(Journey — Before/After 사용자 경험 대비)와 "어떻게 결정하는가"(Flow — 비즈니스 질문 기반 결정 트리). 4개에서 2개로 축소하여 작업기억 부담을 최소화하고(WM-001: 4±1 청크), 구현 디테일(배너 조회 책임 이동, 닫기 피드백 루프)은 맥락 이해에 불필요하여 제외했다. Flow 다이어그램은 세그먼트 분리 + 순차 색칠 애니메이션으로 의사결정 경로를 직관적으로 표현한다.
Applied Rules
  • RULE-CMP02 변경점 5개 이하 유지 — 4개 카테고리를 2개로 축소 ← WM-001
  • RULE-L02 동시 강조 ≤ 2 — 카테고리당 1개 핵심 메시지 ← ATT-002
  • RULE-C01 시맨틱 컬러 — 비회원(주황), 첫 구매(보라), 기존 회원(파랑) 일관된 색상 언어 ← PRC-001
  • RULE-C09 Slate 다크 테마 — Deep Navy 배경 + Slate 카드 ← AES-001
  • RULE-T01 화면당 핵심 메시지 1문장 — category-insight로 프레이밍 ← IP-001
  • EVAL-0002 Before/After 시각적 비대칭 — Before(점선 1개 카드) vs After(실선 3개 카드) ← IP-001
  • I-01 비개발자 10초 이해 — 코드 클래스명 배제, 사용자 여정/비즈니스 질문으로 표현 ← DES-ID-002
  • I-03 사용자 여정 관점 — Flow 카테고리를 코드 호출이 아닌 "로그인했는가? → 구매한 적 있는가?" 비즈니스 질문으로 전환 ← DES-ID-002
