KMA-7359 · 검색 탭 마이그레이션

Fragment 트랜잭션에서
Navigation3 선언적 라우팅으로

상속 기반으로 분산된 검색 네비게이션을 NavDisplay 하나로 통합하여, 화면 전환 로직의 소유권을 명확히 한다

3가지 핵심 변경
  1. 상속 구조 제거 — SearchFragment + SearchInputFragment 2계층 상속을 단일 SearchNavHostFragment로 통합
  2. 선언적 네비게이션 — FragmentManager.commit 트랜잭션을 NavDisplay + NavBackStack으로 전환
  3. 직렬화 가능 라우트 — 도메인 타입 라우트를 @Serializable 프리미티브 타입으로 변환

Cognitive Rationale — 이 시각화가 이 형태인 이유
Pattern
PAT-009 — Categorized Before/After
3가지 독립적 변경(클래스 통합, 네비게이션 패러다임 전환, 라우트 직렬화)이 단일 본질 특성으로 귀결되지 않으므로, 카테고리별 Before/After 대비가 가장 효과적이다. 각 카테고리가 "왜 이 변경이 필요한가"를 독립적으로 전달한다.
Applied Rules
  • RULE-L01 Before/After 그리드가 카테고리 내에서 앵커 포인트 유지 ← WM-001, ATT-002
  • RULE-L02 카테고리 3개 = 작업기억 4±1 한계 내 ← WM-001, WM-002
  • RULE-C01 변경 유형별 색상 일관: merge=주황, move=파랑, emphasis=보라 ← PRC-003
  • RULE-C09 Slate 다크 테마 — Deep Navy 배경 + 카드 표면 위계 ← PRC-004, AES-003
  • RULE-T01 category-insight로 카테고리당 1문장 핵심 메시지 ← IP-001, WM-003
  • RULE-CMP03 concern/neutral 행 대비 + concern-text/flow-note로 변경 안내 ← ATT-004
