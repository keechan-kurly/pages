KMA-6165 · Deep Link

딥링크가 확실하게 열린다

마케팅 캠페인(Branch, AppsFlyer) 딥링크가 조건에 따라 다른 경로로 처리되어 간헐적으로 열리지 않던 문제를 “써드파티 먼저, 실패 시 자체 파서” 단일 체인으로 해결

핵심 변경 요약
⇄

3갈래 분기 → 단일 체인

딥링크 도착 시 “Branch extra가 있는가?”부터 확인하던 3갈래 조건 분기를, 항상 써드파티를 먼저 시도하는 단일 경로로 전환

분기점 3개 → 2개로 감소
−

병렬 실행 → 순차 실행

Branch/Facebook/Dynamic/AppsFlyer 4개를 동시 실행(async)하던 것을 순차 실행으로 변경하여 실행 순서 보장

결과 예측 가능성 향상
↷

AppsFlyer 초기화 시점 정상화

딥링크 처리 후 부수효과로 실행되던 AppsFlyer 초기화를, Branch 결과 획득 직후 올바른 시점으로 이동

Applied Rules
  • RULE-C01 변경 유형별 색상 언어 — 프로세스(보라), 분기(주황), 흐름(초록), 외부(회색) ← PRC-003
  • RULE-C09 Slate 다크 테마 — Deep Navy 배경 + Slate 카드 + 시맨틱 색상 밝기 상향 ← PRC-004, AES-003
  • RULE-L01 앵커 포인트 유지 — Before/After 양쪽에서 Intent 시작 위치 동일 ← WM-001
  • RULE-L02 동시 강조 ≤ 2 — Before 흐름도와 After 흐름도를 섹션 분리하여 한 번에 하나씩 집중 ← PRC-005
  • RULE-L03 관련 변경점 근접 그룹화 — 책임 이동을 별도 섹션으로 그룹화 ← PRC-001
  • RULE-CMP03 변경 안내 제공 — Impact Summary 카드로 3가지 핵심 변경 사전 예고 ← ATT-002
  • RULE-T01 화면당 1문장 — 각 섹션 라벨이 해당 섹션의 핵심 메시지를 1문장으로 전달 ← WM-001
  • RULE-C06 WCAG AA 대비 — 다크 배경 텍스트에 --color-on-surface-body(#94a3b8) 사용 ← AES-004

Evidence Trail
  • L1 Cowan (2001) — 작업기억 용량 4±1 청크. Decision Gate 수가 Before 3개 → After 2개로 감소하여 인지 부하 절감. → RULE-L02
  • L2 Cleveland & McGill (1984) — 위치 인코딩이 면적/각도보다 2-3배 정확. 흐름도 노드 위치로 경로 차이 즉시 전달. → RULE-L04
  • L2 Segel & Heer (2010) — author-driven 시각화가 PR 설명에 최적. Impact Summary → Before → After 순차 안내. → RULE-CMP05
  • L3 Hullman & Diakopoulos (2011) — 서사적 프레이밍 +23% 이해도. "3갈래 분기 → 단일 체인" 메시지 프레이밍. → RULE-CMP03
  • L4 Wickens (2008) — 시선 전환 비용 최소화. Before/After를 섹션 분리하여 컨텍스트 스위칭 감소. → RULE-L01
