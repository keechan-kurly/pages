    04 이벤트 파이프라인
    1
    스크롤 감지
    onScrollStateChanged() / visibleItemsInfo
    RecyclerView OnScrollListener 또는 LazyListState 변화 감지. ScrollImpressionStrategy가 캡처 시점(Idle / Velocity)을 결정.
    t = 0ms
    2
    가시성 계산
    ViewImpressionStrategy.isVisible(view, threshold)
    각 ViewHolder의 getLocalVisibleRect()로 실제 노출 영역 비율 계산. ViewImpressionStrategy가 판정 기준(Ratio / Nested)을 캡슐화.
    t = 0ms
    3
    이벤트 캡처
    ImpressionCapturable.capture() → List<ImpressionTrackUnit>
    임계값(기본 70%) 이상인 ViewHolder만 통과. ImpressionTrackUnit에 itemId, position, 이벤트 파라미터 패킹.
    t = 0ms
    4
    중복 제거
    AnalyticsEventConsumer.isConsumed(identifier)
    세션 내 이미 전송한 identifier는 HashSet으로 제외. 리스트 재사용 시 중복 이벤트 방지.
    t = 0ms
    5
    전송
    AnalyticsEventConsumer.sendEvent(units)
    미소비 TrackUnit 일괄 전송. Amplitude logEvent 호출. markConsumed() 로 identifier 등록.
    t + <1ms
    05 설정값 레퍼런스
    상수 기본값 설명 비고
    VISIBILITY_THRESHOLD 0.7f ViewHolder 가시 비율 임계값 (70%). ViewImpressionStrategy 생성 시 오버라이드 가능. 화면별 조정 가능
    VELOCITY_THRESHOLD 1200 px/s VelocityScrollStrategy 에서 캡처를 허용하는 최대 스크롤 속도. VelocityStrategy 전용
    SCROLL_STATE_IDLE SCROLL_STATE_IDLE IdleScrollStrategy: RecyclerView.SCROLL_STATE_IDLE 진입 시 캡처 트리거. IdleStrategy 전용
    MAX_BATCH_SIZE 없음 (즉시 전송) 현재 구현은 배치 없이 캡처 직후 즉시 전송. 향후 배치 지원 가능. 추후 확장 포인트
