KMA-6272 · WebView Stability · PR #6950

1px의 함정
GPU 텍스처 정렬이 만든 크래시

저사양 단말에서 WebView 크래시가 1만 건을 돌파했다. 원인은 GPU maxTextureSize가 64의 배수가 아닐 때 발생하는 정렬 오버플로우였다.

ROOT CAUSE
LAYER_SIZE(64) 올림 정렬의 함정
GPU 레이어는 64의 배수로 정렬된다. ceil(height / 64) × 64로 계산되는데, maxTextureSize가 16383처럼 64의 배수가 아니면 올림 결과가 한계를 초과한다.

핵심: maxTextureSize = 16383인 디바이스에서 WebView 높이가 16321~16383 구간에 진입하면, 올림 결과 16384가 되어 한계를 1px 초과한다. wrap_content로 설정된 WebView에서 이 구간 진입이 발생했다.

Cognitive Rationale — 시각화 설계 근거

Pattern

PAT-024 (Data Flow Diagram) 선택. 이 이슈의 핵심은 GPU 렌더링 파이프라인 내 데이터 흐름과 분기점이다. 리뷰어의 질문이 "왜 크래시가 나는가? + 어떻게 해결하는가?"이므로 클래스 계층이 아닌 동작/행위를 보여주는 흐름 다이어그램이 효과적이다 (EVAL-0010). Before/After를 좌우 배치하여 경로 분기의 차이를 즉각 대비하도록 설계했다.

Applied Rules

  • RULE-L01 — Before/After 앵커 포인트 유지. 양쪽 시작점(WebView)을 동일 위치에 배치 ← WM-001, ATT-002
  • RULE-L02 — 동시 강조 ≤ 2. 정렬 트랩(빨강) + 해결 흐름(초록) 2가지만 강조 ← PRC-005, WM-002
  • RULE-C01 — 변경 유형별 색상 언어. 위험=빨강, 안전=초록, 흐름=파랑, 판정=보라 ← PRC-003, DM-001
  • RULE-C09 — Slate 다크 테마 기본 적용. Deep Navy 배경 + Slate 카드 ← 기술 문서 톤
  • RULE-T01 — 화면당 한 문장. 각 섹션의 핵심 메시지를 하나로 응축 ← IP-001, WM-003
  • RULE-T02 — 강조 대상 근처 레이블. 각 flow-node에 직접 설명 배치 ← WM-003
  • RULE-CMP03 — 변경 안내. Impact 카드에서 핵심 수치를 사전 예고 후 상세 설명 진입 ← ATT-002
  • RULE-L04 — 시각 인코딩 위계. 디바이스 맵의 바 길이로 maxTextureSize 비교 ← DES-VH-001
