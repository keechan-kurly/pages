Design Approach
4개 카테고리로 변경 사항을 구조화했다. 실시간 삽입 파이프라인(Cat-1)은 데이터 흐름(Flow)과 이를 뒷받침하는 캐시 아키텍처 전환(Journey)을 하나의 카테고리 안에 세로 스택으로 배치했다. 캐시 아키텍처는 삽입 파이프라인의 기반 인프라이므로 독립 카테고리보다 하위 섹션이 자연스럽고, 전체 카테고리 수를 4개로 줄여 WM-001(4±1) 범위 내에 안착시켰다.

스크롤 앵커링(Cat-2)은 Before→After UX 전환이므로 Journey가 적합하다. A/B 테스트(Cat-3)는 독립적인 Feature Flag와 4개 이벤트를 동시에 보여줘야 하므로 Card Grid가 효과적이다. 진입 속도(Cat-4)는 1개 UseCase→2개 UseCase 분리이므로 좌우 비교(Side by Side)가 직관적이다.

카테고리 카드에 accent 그래디언트 틴팅을 적용하여 스크롤 시 현재 카테고리를 색상만으로 즉각 식별할 수 있게 했다. category-insight를 카테고리 타이틀 바로 아래에 배치하여 본문을 읽기 전에 변경 의도를 프레이밍한다.
Applied Rules
  • RULE-L02 4개 카테고리, 각 카테고리 내 동시 강조 요소 ≤ 2. WM-001(4±1) 범위 내에 안착 ← WM-001, WM-002
  • RULE-C01 4개 카테고리에 고유 accent 색상 배정 (파랑/주황/초록/핑크). 범례에서 색상-카테고리 매핑을 명시 ← PRC-003
  • RULE-C03 모든 inner style이 동일한 :root CSS 변수를 공유하여 시각적 다양성과 일관성을 동시에 유지 ← PRC-001
  • RULE-T01 각 카테고리의 cat-insight가 변경 의도를 1문장으로 프레이밍 ← IP-001, WM-003
  • RULE-CMP03 4종 inner style(Flow+Journey, Journey, Card Grid, Side by Side)로 변경 유형이 레이아웃 형태로 인코딩됨 ← ATT-002, ATT-003
  • RULE-C09 Deep Navy 배경(#0a0e1a) + Slate 패널(#1e293b) 기본 다크 테마 적용 ← AES-001
Inner Style Selection Rationale
  • Flow + Journey Cat-1은 5단계 순차 파이프라인(Flow)과 기반 아키텍처 전환(Journey)을 세로 스택으로 합성. Flow가 "어떻게 동작하는가"를 보여주고, 하위 Journey가 "이 동작을 가능하게 한 인프라 변경"을 보여준다. 코드 레퍼런스로 diff 매핑이 즉시 가능 ← ENG-AV-001, DES-ID-002
  • Journey 스크롤 앵커링은 Before(미인지) → After(능동적 유도) UX 전환. 주황색 accent로 Cat-1과 시각적으로 구분 ← PRC-003
  • Card Grid Feature Flag 1개 + 이벤트 4개는 순서가 아닌 집합. 그리드 레이아웃이 독립 항목의 동시 비교를 지원하며, Feature Flag는 전폭으로 계층적 중요도를 인코딩 ← DES-VH-001
  • Side by Side UseCase 분리는 "1개가 2개로 나뉨"이라는 구조적 변화. 좌우 배치에서 Before(단일 박스, 회색 점선) vs After(2개 색상 박스, 실선)의 시각적 밀도 차이가 개선을 즉각 전달 ← DES-CO-001
