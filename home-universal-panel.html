Inner Styles

Journey(API 구조 / 패널 진입 / 이벤트) — Before 단계 vs After 단계 비교로 “무엇이 달라지는가” 직관 전달.

  • RULE-C01 add=#4ade80, remove=#6b7280+취소선, move=#60a5fa, emphasis=#a78bfa, merge=#fb923c — 전체 시각화에서 색상 언어 일관 ← PRC-003
  • RULE-C02 의미 색상 3가지: add(초록), remove(회색), move/emphasis(파랑/보라) — 인지 비용 최소화 ← PRC-005
  • RULE-L02 카테고리당 1개 시각화 스타일 = 1 cognitive chunk, 동시 강조 ≤ 2 ← WM-001, WM-002
  • RULE-T01 각 카테고리 하단 note가 1문장으로 핵심 메시지 전달 ← IP-001, WM-003

  • RULE-S01 모든 간격은 4dp 배수 (8, 16, 24, 32, 48) ← AES-003
Inner Style Selection Rationale
  • Journey (Cat 1) API 호출 구조 — Before 3단계(래퍼+site+헤더 미접근) vs After 3단계(직접호출+panelCode+헤더 추출)로 단순화 과정 직관 전달 ← WM-002
  • Journey (Cat 2) 패널 진입 경로 — Before 6단계(개발 요청~배포) vs After 2단계(CMS 등록~즉시)로 운영 효율 개선을 단계 수 대비로 전달 ← IP-001
