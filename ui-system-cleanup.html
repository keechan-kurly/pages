Cognitive Rationale — 이 시각화의 설계 근거

Pattern

PAT-009 (categorized_before_after) — 3가지 독립적 정비 작업이 하나의 PR에 포함됨. 각 카테고리의 변경 유형이 동일(산재 → 통합)하므로 code-row 기반 Before/After가 적합.

Applied Rules

  • RULE-L01 — Before/After 앵커 정렬: 양쪽 코드 블록의 대상 파일이 동일 위계에 배치 ← PRC-001
  • RULE-L02 — 동시 강조 ≤ 2: 각 카테고리 내 Before/After 2패널만 동시 표시 ← WM-001
  • RULE-C01 — 변경 유형별 색상: concern(주황), old(회색+취소선), new(초록), merged(주황), neutral(점선) ← PRC-003
  • RULE-C09 — Slate 다크 테마: Deep Navy 배경 + Slate 카드 ← AES-001 (처리 유창성)
  • RULE-T01 — 화면당 한 문장: category-insight에 1~2문장 변경 의도 요약 ← IP-001
  • RULE-CMP03 — 변경 안내: concern 행으로 Before의 문제점을 사전 예고 ← ATT-003
  • RULE-C06 — 다크 테마 대비: Before 텍스트에 --color-on-surface-body(#94a3b8) 사용 ← WCAG 2.1 AA

Self-Evaluation

  • cognitive_load: 2/5 — 카테고리 3개, 각 카테고리 내 변경점 2~5개
  • change_visibility: 5/5 — concern/neutral 대비 + 색상 코딩 + category-insight + 그래디언트 틴팅
  • anchor_stability: 5/5 — Before/After 동일 grid 배치, 파일→파일 대응
  • text_economy: 5/5 — category-insight 1~2문장, code-row 내 핵심만

Evidence Trail

  • L2 PRC-001 — 전주의적 속성 인코딩: 색상·형태로 코드 변경 유형 즉시 구분 (Healey & Enns, 2012)
  • L2 WM-001 — Miller's 4±1 법칙: 카테고리 3개는 작업기억 용량 내 (Cowan, 2010)
  • L2 ATT-003 — 변화 맹시 방지: concern 행이 Before의 문제를 사전 표시 (Rensink et al., 1997)
  • L3 IP-001 — 이중 부호화: 시각(색상/레이아웃) + 언어(insight 텍스트) 동시 전달 (Paivio, 1986)
  • L4 AES-001 — 처리 유창성: Slate 다크 테마로 시각적 피로 감소 (Reber et al., 2004)
